# Traceability

Traceability is the machine-checkable chain that proves every requirement is implemented and tested, and that nothing is
orphaned. It links **spec clause → story → acceptance criterion → test → module**. The chain is materialized in
`docs/traceability.yaml`, a **generated** file, and validated by `./gradlew traceCheck`.

## the-chain {#the-chain}

```
spec clause ──cited by──► story ──declares──► acceptance criterion ──proven by──► test ──lives in──► module
   (FR-…/anchor)            (STORY-NNN)          (STORY-NNN-AC-N)        (// Proves:)      (:module/pkg)
```

Every link must close in both directions:

- Each **story** cites at least one resolvable spec clause and at least one module from
  `docs/implementation_plan/01_MODULE_INVENTORY.md`.
- Each **acceptance criterion** is proven by at least one test.
- Each **cited edge case** (`EC-…`) is covered by at least one test.
- Each **spec clause** in an **implemented** phase's `phase_clauses:` manifest is cited by some non-superseded story (no
  orphan clause — checked at phase exit, `#orphan-clause-phase-exit`).
- No **test** claims a `Proves:` id that no story declares (no orphan test).

## proves-convention {#proves-convention}

A proving test declares the acceptance criterion (or edge case) it proves on its **first line**, as a Java comment
immediately inside the test method:

```java
@Test
void detectsContentLanguageOverMetadata() {
    // Proves: STORY-001-AC-1
    ...
}
```

Rules:

- The marker is a `// Proves: STORY-NNN-AC-N` line comment. Exactly this spelling (`Proves:` capitalized, single space).
  One marker per line; a test may carry more than one `// Proves:` line if it proves several ACs.
- Edge cases are declared the same way: `// Proves: EC-LANG-1`. An edge-case marker must correspond to an `edge_cases`
  entry in some story's front-matter.
- The marker is the **first statement** in the test body (before arrange/act/assert), so the generator can find it
  without parsing logic. (There is no Jest-style `it('...')` convention here — this is a JUnit 5 codebase; the comment
  marker is the single mechanism across unit, integration, UI, and arch tiers.)

## traceability-yaml-schema {#traceability-yaml-schema}

`docs/traceability.yaml` is generated by `./gradlew trace` and must never be hand-edited. Schema (four top-level maps):

```yaml
stories:
  STORY-001:
    title: Detect real source language ignoring declared metadata
    status: done
    phase: PHASE_01_DOCUMENT_MODEL
    spec_clauses: [FR-IMPORT-03, docs/specification/01_Product/03_DOCUMENT_FORMATS.md#drm-and-language-detection]
    modules: [":document/ua.bookloom.document.detect", ":util/ua.bookloom.util.lang"]
    depends_on: []
    acceptance_criteria:
      STORY-001-AC-1:
        proven_by:
          - test: ua.bookloom.document.detect.LanguageDetectorTest#detectsContentLanguageOverMetadata
            file: document/src/test/java/ua/bookloom/document/detect/LanguageDetectorTest.java
            tier: unit
            module: ":document/ua.bookloom.document.detect"
      STORY-001-AC-2:
        proven_by: [ ... ]
    edge_cases:
      EC-LANG-1:
        proven_by:
          - test: ua.bookloom.document.detect.LanguageDetectorTest#flagsMismatchWhenMetadataDisagrees
            file: document/src/test/java/ua/bookloom/document/detect/LanguageDetectorTest.java

clauses:
  FR-IMPORT-03:
    resolves_to: docs/specification/01_Product/01_FUNCTIONAL_REQUIREMENTS.md#fr-import
    cited_by: [STORY-001]

edge_cases:
  EC-LANG-1:
    defined_in: docs/specification/01_Product/03_DOCUMENT_FORMATS.md#drm-and-language-detection
    cited_by: [STORY-001]
    proven_by: [ua.bookloom.document.detect.LanguageDetectorTest#flagsMismatchWhenMetadataDisagrees]

modules:
  ":document/ua.bookloom.document.detect":
    inventory: present
    touched_by: [STORY-001]

meta:
  generated_at: 2026-07-17T00:00:00Z
  source_fingerprint: <canonical hash over sorted (story, AC, test) tuples + cited-clause and module sets>
```

The `source_fingerprint` is a **canonical** hash over the **sorted `(story, AC, test)` tuples** (plus cited-clause and
module sets), so it is stable under incidental reordering. Note the done-immutability check (row 12) uses a **narrower**
scope — the AC set and per-AC Proves-coverage existence, not exact test-method identity — so a test refactor that
preserves coverage does not trip immutability even though it changes this record's `test` FQNs (`#check-table`).

- **`stories{}`** — one entry per story, its metadata, and, per AC/edge case, the tests that prove it.
- **`clauses{}`** — every cited clause, where it resolves, and which stories cite it.
- **`edge_cases{}`** — every cited `EC-`, where defined, cited by, and proven by.
- **`modules{}`** — every module referenced, whether it is present in the inventory, and which stories touched it.
- **`meta`** — generation timestamp and a `source_fingerprint` used by `traceCheck` to detect a stale record.

## gradle-tasks {#gradle-tasks}

| Task                   | Action                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                          |
|------------------------|-----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| `./gradlew trace`      | **Regenerate** `docs/traceability.yaml`. Concretely: a **JavaParser** pass walks **every test source set** (unit, integration, UI, arch) and maps each `// Proves: <id>` marker to the **fully-qualified name of its enclosing test method** (`pkg.Class#method`); a **parsed FR-ID index** is built by scanning the spec requirement tables (via `docs/specification/00_Foundation/05_SPEC_INDEX.md`) so every `FR-…`/`NFR-…`/`DD-…`/`EC-…`/anchor a story cites can be resolved; story front-matter/bodies and the module inventory are scanned; and a **canonical `source_fingerprint`** is written as a stable hash over the **sorted `(story, AC, test)` tuples** (plus the cited-clause and module sets). |
| `./gradlew traceCheck` | **Validate** the record against the checks below. Fails the build (and blocks the DoD) on any violation. Recomputes the fingerprint and fails if the record is stale. The **orphan-clause check runs at phase exit**, not per story (see `#orphan-clause-phase-exit`).                                                                                                                                                                                                                                                                                                                                                                                                                                          |

Until the scaffold story (PHASE_00) implements these tasks, the same checks are maintained by review against this
schema.

## orphan-clause-phase-exit {#orphan-clause-phase-exit}

The "no orphan clause" check (row 3) is a **phase-exit** check, **not** a per-story check — this removes the deadlock
where a phase's clauses would be reported as orphaned while its stories are still in progress (and no single story can
satisfy every clause of its phase on its own).

- A phase is an **"implemented phase"** only once **all of its stories are `done`**. Until then its clauses are **out of
  scope** for the orphan check and cannot fail it.
- Each phase file carries a machine-readable **phase→clause manifest** as YAML front-matter (`phase_clauses:` — the list
  of spec clauses that phase is responsible for). `traceCheck` reads these manifests; the orphan check asserts that, for
  every **implemented** phase, each clause in its manifest is cited by at least one non-superseded story.
- A clause belonging only to a not-yet-implemented phase is never an orphan. This makes the clean-gate satisfiable
  during parallel/early phases while still guaranteeing full coverage by the time a phase closes.

## check-table {#check-table}

`./gradlew traceCheck` runs the following. Each row states the check and the condition that **fails** it.

| #  | Check                             | Fails when                                                                                                                                                                                                                                                                                                                                                                                                                                                             |
|----|-----------------------------------|------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| 1  | Clause resolves                   | A story's `spec_clauses` entry does not resolve to a known spec anchor or requirement id (via `docs/specification/00_Foundation/05_SPEC_INDEX.md`).                                                                                                                                                                                                                                                                                                                    |
| 2  | Module exists                     | A story's `modules` entry is absent from `docs/implementation_plan/01_MODULE_INVENTORY.md`.                                                                                                                                                                                                                                                                                                                                                                            |
| 3  | No orphan clause (**phase-exit**) | For an **implemented phase** (all its stories `done`), a clause in that phase's `phase_clauses:` manifest is cited by no non-superseded story. Not evaluated for phases still in progress (`#orphan-clause-phase-exit`).                                                                                                                                                                                                                                               |
| 4  | No orphan story                   | A story cites a clause that does not exist, or has an empty `spec_clauses`/`modules`.                                                                                                                                                                                                                                                                                                                                                                                  |
| 5  | No orphan test                    | A `// Proves: STORY-NNN-AC-N` (or `EC-…`) marker names an AC/edge case that no story declares.                                                                                                                                                                                                                                                                                                                                                                         |
| 6  | AC proven                         | An acceptance criterion listed in a story has no proving test.                                                                                                                                                                                                                                                                                                                                                                                                         |
| 7  | Edge case covered                 | An `EC-…` in a story's `edge_cases` has no proving test.                                                                                                                                                                                                                                                                                                                                                                                                               |
| 8  | Acyclic deps                      | The `depends_on` graph across stories contains a cycle.                                                                                                                                                                                                                                                                                                                                                                                                                |
| 9  | Record fresh                      | The committed `docs/traceability.yaml` `source_fingerprint` does not match the current stories/tests/inventory (i.e. `trace` was not re-run after a change).                                                                                                                                                                                                                                                                                                           |
| 10 | AC set matches                    | A story's `acceptance_criteria` front-matter does not equal the set of `### STORY-NNN-AC-N` body headings.                                                                                                                                                                                                                                                                                                                                                             |
| 11 | ADR exists                        | A story's `adrs` entry has no file under `docs/adr/`.                                                                                                                                                                                                                                                                                                                                                                                                                  |
| 12 | Done immutability                 | A `done` story's **AC set or the existence of `Proves:` coverage for any AC** changed without a superseding story. The check is scoped to the **AC set + per-AC Proves-coverage existence**, **not** exact test-method identity — a pure test refactor (rename/move a proving method while every AC keeps at least one `Proves:` marker) does **not** trip it and does **not** force a superseding story. Dropping an AC, or leaving an AC with no proving test, does. |
| 13 | Phase resolves                    | A story's `phase` value (`PHASE_NN_SLUG`) does not resolve to a file `docs/implementation_plan/phases/<phase>.md`.                                                                                                                                                                                                                                                                                                                                                     |

A story is not `done` (see `docs/implementation_plan/06_DEFINITION_OF_DONE.md`) until `traceCheck` passes with zero
violations and a fresh record.

## authoring-workflow {#authoring-workflow}

1. Architect writes the story with `spec_clauses`, `modules`, `acceptance_criteria`, `edge_cases`.
2. Coder/tester implement production code and tests; each test carries its `// Proves:` marker as its first line.
3. Run `./gradlew trace` to regenerate the record, then `./gradlew traceCheck` to validate.
4. Fix any orphan/coverage/freshness violation and re-run until green.
5. Mark the story `done` (immutable thereafter).
