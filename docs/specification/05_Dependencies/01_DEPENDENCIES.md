# Dependencies

The concrete library set, why each is chosen, and which module owns it. All coordinates/versions live in
`gradle/libs.versions.toml`; exact pins and the policy are in `02_DEPENDENCY_POLICY.md`. Every library is
permissive-licensed (`03_LICENSING.md`).

## test-libraries {#test-libraries}

| Library              | Purpose           | Why                                                                                                                                                                                      |
|----------------------|-------------------|------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| **JUnit 5**          | test framework    | standard JVM testing                                                                                                                                                                     |
| **AssertJ**          | fluent assertions | readable, rich assertions incl. records                                                                                                                                                  |
| **Mockito 5**        | mocking           | isolate ports/collaborators                                                                                                                                                              |
| **WireMock**         | HTTP stub         | the LLM HTTP seam — deterministic provider behavior, error/Retry-After simulation, and an egress deny-default for the offline test                                                       |
| **TestFX + Monocle** | headless UI test  | drive JavaFX in CI with no display server (`04_Build_and_Release/04_CI_CD.md`)                                                                                                           |
| **JaCoCo**           | coverage          | exact **80% branch, per-module, on modules with production code only** (empty/stub and `:ui`/`:app` presentation excluded); see `04_Build_and_Release/02_QUALITY_GATES.md#coverage-gate` |
| **PIT** (optional)   | mutation testing  | strengthen `:document`/`:pipeline` tests                                                                                                                                                 |

No Testcontainers — SQLite is embedded; tests use a temp DB file or `:memory:`.

## ownership-note {#ownership-note}

Each library is declared only in the module (s) that need it (narrowest scope, `02_DEPENDENCY_POLICY.md`).
`java.net.http` stays in `:llm`; `java.sql`/JDBI/Flyway stay in `:persistence`; `javafx.*`/AtlantaFX/Ikonli/ControlsFX
stay in `:ui`/`:app` — mirroring the JPMS `requires` in `02_Architecture/02_MODULES_AND_LAYERING.md`.
