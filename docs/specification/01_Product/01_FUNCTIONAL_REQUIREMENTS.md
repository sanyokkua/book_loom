# Functional Requirements

This is the master, gap-free catalog of functional requirements, one table per requirement area. Each requirement is
atomic and testable. Priority is Must / Should / Could. Detailed behaviour for most areas is elaborated in the
area-specific spec documents cross-referenced above; those documents cite these IDs.
