# Specification Clause Index

This is the canonical index of citable clauses. Stories, ADRs, and commands resolve citations as `<file>#<anchor>`,
where `<file>` is repo-relative under `docs/specification/`. Every anchor below is a stable kebab-case heading anchor in
the named file.

## how-to-cite {#how-to-cite}

Cite a clause as `docs/specification/<path>#<anchor>` (e.g.
`docs/specification/01_Product/01_FUNCTIONAL_REQUIREMENTS.md#fr-algo`). Requirement IDs (`FR-…`, `NFR-…`, `DD-…`,
`EC-…`) are cited by ID and resolved to the file/anchor where they are defined. This index lists the section-level
anchors; the requirement tables under each anchor carry the individual IDs.
