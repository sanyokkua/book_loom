# LLM Provider Integration

Scope: `:llm` (`ua.bookloom.llm..`). Spec: `docs/specification/02_Architecture/04_LLM_INTEGRATION.md`. Transport is JDK `java.net.http.HttpClient`; JSON is Jackson over records. FX-free.

## MUST

- **MUST** expose provider access only through the `Provider` port + `ProviderFactory`; the concrete client is hidden from callers, who never branch on kind. Two implementations exist: an **Ollama-native** client (`/api/chat`, `/api/tags`, `/api/show`, native `options`) for `OLLAMA`, and an **OpenAI-compatible** client (`/v1/chat/completions`, `/v1/models`, `response_format`) for `OPENAI_COMPATIBLE` (LM Studio and any other OpenAI-shaped server). The kind enum is extensible; a future provider (Gemini/Claude) is a new implementation registered in the factory ŌĆö not implemented here. ŌĆö Rationale: callers stay dialect-agnostic; the seam is open without speculative code.
- **MUST** provide a dedicated `probe()`/`verifyConnection()` on the `Provider` port (Ollama root/`/api/version`; OpenAI-compatible `/v1/models` HEAD/cheap GET) so the **Connection and Models verification stages are separable** ŌĆö a server with no discovery endpoint still passes Connection independently. ŌĆö Rationale: reachability is provable without listing models or running inference.
- **MUST** keep `chat` **synchronous, returning `Result<ChatResponse>`** (whole body). **Streaming is out of scope for v1** ŌĆö there is no `ChatRequest.stream` field and no streaming path. Cancellation = the HTTP request timeout (hard bound) + a cooperative interrupt; worst-case cancel latency is the request read-timeout, not instant. ŌĆö Rationale: v1 has no incremental-render need; a bounded synchronous call is simpler and still cancellable.
- **MUST** talk to Ollama through its **native API**, not the OpenAI-compatible shim, so `num_ctx`, `keep_alive`, `format`, and `think` are actually honored ŌĆö the `/v1/*` surface does not fully apply them. ŌĆö Rationale: reliable context-window and reasoning/format control for local Ollama models.
- **MUST** run every model-facing response through the JSON-first tolerant pipeline: **always attempt** structured output (OpenAI `response_format` / Ollama `format`), set reasoning low/off where controllable, **sanitize** (strip `<think>` blocks, chain-of-thought preambles, code-fence wrappers, stray prose), **parse tolerantly** (unknown ignored, missing defaulted, trimmed), do **one** repair retry on malformed JSON, then fall back to the sanitized text as the plain translation ŌĆö flagging only if QA then fails. ŌĆö Rationale: models wrap/narrate JSON; be strict in the ask, tolerant in the parse.
- **MUST** treat a **structured-output rejection (HTTP 400 / unsupported) as a silent downgrade**: re-issue the call as a plain request and continue down the text-fallback path ŌĆö never a hard failure ŌĆö and record the result on a `supportsStructuredOutput` capability that is **populated during the Models/Inference verification stage**. ŌĆö Rationale: capability is discovered, not assumed; a picky server degrades gracefully.
- **MUST** keep reasoning control **best-effort**: set the reasoning/effort parameter low/off where controllable (Ollama `think`, informed by `/api/show`; OpenAI-compatible reasoning-effort), **omit it when unsupported**, and always fall through to the `<think>`-strip sanitize path regardless. ŌĆö Rationale: some models can't disable reasoning; the strip path is the guarantee.
- **MUST** order empty responses distinctly: a **raw-empty** (blank *pre-sanitize*) response ŌåÆ `emptyCompletion`; a **non-empty response that sanitizes to empty** ŌåÆ **one repair retry** specific to empty-after-sanitize, then an empty text-fallback result that **QA flags** (not `emptyCompletion`). ŌĆö Rationale: "model emitted nothing" and "model emitted only reasoning" are different failures.
- **MUST** construct both clients with **one injected `java.net.http.HttpClient`**, a **fresh per-request timeout**, request/response DTOs as **records in an internal `dto` package** (`@JsonInclude(NON_NULL)`), a **tolerant `ObjectMapper`**, and **never log secrets or full book text**. ŌĆö Rationale: shared transport, no leaked prompts/credentials, no explicit-null rejections.
- **MUST** verify connection + bound-model availability **before ANY inference** (real runs and diagnostics), failing fast with a typed error; a project **reuses its own bound provider/models**, and any fallback to the current default happens **only on user confirmation** ŌĆö never a silent switch. `current_provider` is the **new-project default only** and is **never consulted for an existing project's run**. ŌĆö Rationale: no mid-run quality regression from a moved-out-from-under-it provider.
- **MUST** define **"available"** as: with discovery, membership in `listModels()`; without discovery (a manual model ID), the **Inference round-trip is the availability check** (`modelUnavailable` on failure). Preflight the **translator model always**, and the **judge/helper model only when the quality dial will invoke the judge**. ŌĆö Rationale: availability has no meaning beyond "the list, or a real call"; an unused judge slot needn't be proved.
- **MUST**, on resume, compare the project's `last_used_json` against the **live provider row** and, on **any drift** (not only a settings-default difference), raise the ADR-0012 "apply new vs continue" prompt (default: continue). ŌĆö Rationale: any silent divergence between snapshot and live config is a surprise the user must confirm.
- **MUST** run a **credential-resolution pre-check** before the Connection probe, mapping a not-yet-resolvable draft credential to `missingCredential` and **failing the Connection stage** ŌĆö no network round-trip against a missing secret. Diagnostics acquire the gate via `tryRun` (bounded wait) and surface a held gate as `busy` ("model in use, try again" toast). ŌĆö Rationale: a draft with an unresolved secret fails fast and locally.
- **MUST** omit nullable `ChatRequest` parameters from the serialized JSON entirely (Jackson non-null inclusion) rather than sending `null`. ŌĆö Rationale: some servers reject explicit nulls; absent means "use default".
- **MUST** store credentials **as a reference only** ŌĆö an env-var name or an OS-keychain handle ŌĆö and resolve the secret at call time. The secret value is **never** persisted, logged, or placed in an `AppError`. ŌĆö Rationale: privacy invariant; the app holds a pointer, not the key.
- **MUST** serialize inference through a single-flight `InferenceGate` backed by `Semaphore(1)`; `run` blocks, `tryRun` uses `tryAcquire` and a failed acquire returns `AppError(busy, retryable=false)` rather than blocking/queuing. The gate is **released during backoff / `Retry-After` sleeps and re-acquired per attempt**, so a sleeping call never holds the permit. ŌĆö Rationale: a local model serves one request at a time; the gate prevents overload without starving on sleeps.
- **MUST** make the **repair retry a fresh call**: it **re-acquires the gate**, gets its **own fresh timeout**, is subject to **transport retry independently**, **does not recurse** into a second repair, and is bounded ŌĆö together with transport retry ŌĆö by a **single global attempt cap** (transport-retry ├Ś repair can't run away). ŌĆö Rationale: repair is a re-ask, not a nested loop; one cap keeps total work finite.
- **MUST** own retry inside the service, keyed on **typed retryable** `ErrorCode`s (`timeout`/`rateLimited`/`unreachable`/`upstream`), honouring `Retry-After` and using a **fresh timeout per attempt**; non-retryable codes fail immediately. ŌĆö Rationale: retries are principled and bounded, not blind.
- **MUST** map every HTTP outcome to a typed `AppError` (`401/403ŌåÆauth`, `429ŌåÆrateLimited`, connect-refusedŌåÆ`unreachable`, `5xxŌåÆupstream`, unknown modelŌåÆ`modelNotFound`, `200`-but-raw-emptyŌåÆ`emptyCompletion`, over-contextŌåÆ`contextWindow`, discovery failureŌåÆ`discoveryFailed`, model unavailable at preflight/runŌåÆ`modelUnavailable`, unresolved credentialŌåÆ`missingCredential`). A structured-output rejection is **not** an error (silent downgrade). ŌĆö Rationale: callers branch on codes; see `error-envelope.md`.

## SHOULD

- **SHOULD** support the three-stage verification against the **draft** provider config before it is saved: **Connection** via the dedicated `probe()` (separable from listing), **Models** via `listModels()` (`discoveryFailed` on hard failure), **Inference** via a tiny `chat` round-trip that also **populates `supportsStructuredOutput`**. ŌĆö Rationale: users validate a provider before committing it, and each stage is independently reportable.
- **SHOULD** carry a per-provider **effective context (tokens)** value resolved **Ollama `num_ctx`/`/api/show` ŌåÆ discovery ŌåÆ manual "effective context (tokens)" field ŌåÆ conservative default**, and have the chunk packer budget `min(effectiveContext ŌłÆ reservedHeadroom, chunkBudgetSetting)`. Colliding `numCtx` inputs resolve **request > provider > setting**; the packer budgets the single resolved value. ŌĆö Rationale: token budgeting needs a real per-provider window, not a guess.
- **SHOULD** treat **manual model-ID entry as first-class and always available**: carry `ProviderProfile.supportsModelDiscovery`, and when discovery is absent/fails/empty/unauthorized, use free-text model-ID entry (pre-filled with the remembered model). Only translator + judge/helper slots exist ŌĆö no embedding slot. ŌĆö Rationale: not every server lists models; consistency comes from dictionary + string-similarity TM + rolling summary, not vectors.
- **SHOULD** keep DTOs as records in a non-exported `dto` package `opens` to Jackson only. ŌĆö Rationale: wire types stay internal and records-first.

## Reject if

- A caller branches on provider kind or talks to a concrete client directly instead of going through the `Provider` port + `ProviderFactory`.
- Ollama is driven through the OpenAI-compatible `/v1/*` shim instead of its native API, or `num_ctx`/native-only options are forced onto an OpenAI-shaped request instead of the Ollama-native client.
- Model output is parsed strictly (no sanitize / no tolerant parse), repair-retried more than once, the repair recurses into a second repair, or output is hard-failed instead of falling back to the sanitized plain text.
- A structured-output rejection (400/unsupported) is treated as a hard failure instead of a silent downgrade to a plain request + text fallback.
- A `ChatRequest.stream` field or any streaming path is (re)introduced, or `chat` is made asynchronous/partial instead of returning the whole `Result<ChatResponse>`.
- A non-empty-but-sanitizes-to-empty response is reported as `emptyCompletion` instead of taking its one empty-after-sanitize repair and then flagging via QA.
- The `InferenceGate` is held across a backoff / `Retry-After` sleep instead of being released and re-acquired per attempt.
- The judge/helper model is preflighted when the quality dial will not invoke the judge, or drift between `last_used_json` and the live provider row is applied without the ADR-0012 confirmation prompt.
- `current_provider` is consulted for an existing project's run instead of only as the new-project default.
- Inference runs against a project without preflight-verifying its bound provider/model, or a provider/model is silently switched without user confirmation.
- An embedding model slot, vector store, or vector-based similarity is introduced.
- A secret value is persisted, logged, put in `AppError`, or stored anywhere but as an env-var/keychain reference; or full book text / prompt bodies reach a log.
- Nullable `ChatRequest` params are serialized as explicit `null`.
- Inference runs without acquiring the `InferenceGate`, or the gate blocks/queues instead of returning `busy`.
- Retry fires on a non-retryable code, ignores `Retry-After`, or reuses one deadline across attempts.
- An HTTP status is surfaced raw instead of mapped to an `ErrorCode`.
